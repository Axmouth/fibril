Settings

Runtime settings apply live and, in a cluster, replicate to every broker - no restarts. Startup-only values live in the configuration file.

Persisted settings are applied live where supported. Broker runtime settings and Stroma-owned dead-letter defaults are saved separately, each with its own version check.

Broker Runtime Settings

Live broker behavior such as delivery timing and idle queue cleanup.

Delivery

Controls how long messages can stay leased and how aggressively delayed or expired work is polled.

Connections

Controls how the broker treats short TCP disconnects from clients that can resume.

Replication

Controls replica-durable publish confirms and follower catch-up work.

Partitioning and Consumer Groups

Defaults for new partitioned queues and exclusive consumer-group health signals.

Idle Queue Cleanup

Lets unused queues leave memory after they have no active publishers or subscribers for long enough.

Applies the Broker Runtime Settings above, live - no restart. Startup Config below is read-only and set before the broker starts.
Desktop Notifications

Notify this browser when a new attention condition appears - a quarantined partition, a backlog with no consumer, a certificate near expiry. Stored in this browser only; the broker is not involved.

Users

Broker accounts for client authentication. The built-in fibril/fibril pair works from loopback only, so create a user for remote access. Passwords are stored as argon2 hashes and never shown.
